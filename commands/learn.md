# /learn - 提取可重用模式

分析当前会话并提取任何值得作为技能保存的模式。

## 触发条件

在会话期间的任意时刻，当你已解决一个非平凡问题时，运行 `/learn`。

## 待提取内容

寻找以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 是什么解决了这个问题？
   - 这是否可用于类似错误？

2. **调试技巧**
   - 不明显的调试步骤
   - 生效的工具组合
   - 诊断模式

3. **变通方法**
   - 库的特殊行为
   - API 的限制
   - 版本特定的修复方法

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建一个技能文件：

```markdown
# [描述性模式名称]

**提取时间：** [日期]
**上下文：** [简要描述适用情况]

## 问题
[此模式解决的问题——具体说明]

## 解决方案
[该模式/技术/变通方法]

## 示例
[如有适用，提供代码示例]

## 使用场景
[触发条件——什么情况应激活此技能]
```

## 流程

1. 回顾会话中可提取的模式
2. 识别最有价值/可重用的见解
3. 撰写技能文件
4. 在保存前请求用户确认
5. 保存至 `~/.claude/skills/learned/`

## 注意事项

- 不要提取琐碎的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定 API 中断等）
- 聚焦于将在未来会话中节省时间的模式
- 保持技能聚焦——每个技能一个模式
