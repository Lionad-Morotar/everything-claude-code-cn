# 代码审查

对未提交更改的综合安全与质量审查：

1. 获取更改文件：git diff --name-only HEAD

2. 对每个更改文件，检查以下内容：

**安全问题（严重）：**
- 硬编码凭据、API 密钥、令牌
- SQL 注入漏洞
- XSS 漏洞
- 缺失输入验证
- 不安全的依赖项
- 路径遍历风险

**代码质量（高）：**
- 函数超过 50 行
- 文件超过 800 行
- 嵌套深度超过 4 层
- 缺失错误处理
- console.log 语句
- TODO/FIXME 注释
- 公共 API 缺失 JSDoc

**最佳实践（中）：**
- 可变模式（建议使用不可变对象）
- 代码/注释中使用表情符号
- 新代码缺失测试
- 可访问性问题（a11y）

3. 生成报告，包括：
   - 严重程度：严重、高、中、低
   - 文件位置与行号
   - 问题描述
   - 建议修复方案

4. 若发现严重或高优先级问题，则阻止提交

切勿批准存在安全漏洞的代码！
